## 현재 구현 완료 여부

MVP 첫 플레이 루프 구현은 완료 상태로 판단한다.

구현은 고정된 첫 가문과 초대 인물로 시작해 3개 사건을 순서대로 진행하고, 선택 결과를 결과 문구, 최소 상태 변화, 연대기 항목으로 피드백한 뒤 세대 요약을 표시한다.

## 제품 완료 기준별 충족 여부

| 제품 완료 기준 | 상태 | 근거 |
|---|---|---|
| 새 실행을 시작할 수 있다 | 충족 | 시작 화면과 첫 사건 진입 버튼 구현 |
| 고정된 첫 가문과 초대 인물의 시작 정보를 볼 수 있다 | 충족 | `하서가`, `하서 단`, 도입 문장 표시 |
| 3개의 사건을 순서대로 진행할 수 있다 | 충족 | 스모크와 수동 GUI에서 사건 1~3 진행 확인 |
| 각 사건에서 2개 이상의 선택지를 고를 수 있다 | 충족 | 각 사건에 선택지 2개 구현 |
| 선택 후 결과 문구와 최소 상태 변화를 볼 수 있다 | 충족 | 결과 화면과 상태 패널 갱신 구현 |
| 선택 결과가 연대기 항목으로 누적된다 | 충족 | `RunState.ApplyChoice`가 `ChronicleEntry` 누적 |
| 3개의 사건 후 세대 요약을 볼 수 있다 | 충족 | 누적 연대기, 최종 상태, 마무리 문장 표시 |
| Project Claude가 지정된 확인 명령을 실행하고 결과를 보고할 수 있다 | 충족 | 구현 보고서 작성 및 검증 결과 기록 |

다음 항목은 PRD와 current-focus에서 제외 범위로 둔 항목이며 이번 완료 판단의 미구현 제약으로 남긴다.

- 완성형 가문 생성기

- 첫 MVP 사건들의 최종 문안 확정
- 복잡한 후계, 혼인, 혈통 시스템
- 랜덤 사건 선택
- 저장/불러오기
- 밸런스 수치 확정
- 완성형 UI/아트 스타일 확정
- CI, 배포, 플랫폼 패키징

추가 구현 메모:

- `MvpLoopContent.cs`의 상태 키, 상태 표시명, 가문명, 인물명, 도입 문장, 사건 문안은 모두 MVP 검증용 임시 값이다.
