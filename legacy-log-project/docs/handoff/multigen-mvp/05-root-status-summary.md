## 현재 구현 완료 여부

다세대 MVP 구현은 Project Claude 보고 기준으로 완료 상태로 판단한다.

구현은 기존 단일 세대 첫 루프를 한 세대 실행기로 유지하면서, 1세대 종료 후 후계가 있는 경로로 2세대에 진입하고, 2세대 종료 후 후계 부재와 사망/퇴장 조건으로 가문 멸문을 확인하는 흐름을 추가했다.

자동 검증은 통과했다. 수동 GUI 확인도 사용자 확인 기준으로 완료됐다.

## 제품 완료 기준별 충족 여부

| 제품 완료 기준 | 상태 | 근거 |
|---|---|---|

| 단일 세대 사건 루프가 유지된다 | 충족 | 기존 3개 사건과 선택 결과/상태 변화/연대기 누적 흐름 유지 |
| 선택 후 결과 문구와 최소 상태 변화를 확인할 수 있다 | 충족 | `ShowResult`에서 결과 문구, `StateDelta`, 최신 연대기 항목 표시 |
| 선택 결과가 연대기에 누적된다 | 충족 | `RunState.Chronicle`에 현재 세대 항목식 기록 누적 |
| 세대 종료 후 세대 요약을 볼 수 있다 | 충족 | `ShowGenerationSummary`와 `GenerationRecord.SummaryParagraph` 구현 |
| 세대 요약이 후대 기록자 시점 요약을 포함한다 | 충족 | 요약 단락에 유산, 다음 세대 암시, 세대 말 상태 서술 통합 |
| `가문력 N년` 형식의 시간 표시가 보인다 | 충족 | 세대 헤더와 요약 헤더에 표시 |
| 후계가 있으면 다음 세대로 진행할 수 있다 | 충족 | 1세대 `NaturalDeath` 후 후계 후보를 선택해 2세대 진입 |
| 최소 2세대 이상을 관찰할 수 있다 | 충족 | 스모크가 1세대와 2세대 모두 진행 |
| 후계 부재와 사망/퇴장 조건으로 멸문을 확인할 수 있다 | 충족 | 2세대 `Extinction` 경로와 멸문 화면 구현 |
| 가문 상태가 다음 세대로 이어진다 | 충족 | `FamilyRunState.FamilyStats`가 진실 출처로 유지되고 세대 전환 시 자리표시자 거동 적용 |
| 큰 사건 결과가 다음 세대 입력으로 이어진다 | 충족 | `EventChoice.OutcomeTag`를 누적해 `CarriedOutcomeTags`로 유지 |
| 연대기가 다음 세대로 이어진다 | 충족 | 끝난 세대는 `FamilyRunState.History`에 `GenerationRecord`로 누적 |
| 상태 기본 표시는 질적 라벨을 사용한다 | 충족 | 4단계 자리표시자 구간 라벨 표시 |
| 추가 정보에서 숫자와 변화 방향을 볼 수 있다 | 충족 | 상세 토글 영역과 `BuildDetailedStateLines` 구현 |
| 특성 시스템은 최소 스텁으로만 도입된다 | 충족 | `TraitStub`은 표시 전용이며 계산 입력으로 쓰지 않음 |

## 검증 결과 요약

Project Claude 보고 기준:

- `dotnet build`: 성공, 경고 0, 오류 0
- `./scripts/godot.sh --headless --import`: 성공
- `./scripts/godot.sh --headless -- --smoke`: 성공, 종료 코드 0, 마지막 줄 `SMOKE_OK`

스모크는 한 실행에서 다음을 확인했다.

- 1세대 사건 3개 진행
- 1세대 `NaturalDeath` 종료
- 후계 후보 확인 및 선택
- 2세대 진입
- 이전 세대 상태, 결과 태그, 기록 이력 이어짐
- 2세대 사건 3개 진행
- 2세대 `Extinction` 종료
- 가문 멸문 출력

## 남은 제약 또는 미구현 범위

다음은 의도적으로 미구현 또는 자리표시자로 남긴 범위다.

- 후계 가능성 계산식은 확정하지 않았다.
- 상태 감쇠율, 상태 구간 임계값, 상태 키 셋은 확정하지 않았다.
- `StateCarryMap`, `BuildStateBandLabel`, `PlaceholderEvaluateSuccession`, `ApplyPlaceholderStateCarry`는 MVP 검증용 임시 로직이다.
- 특성 스텁은 표시 전용이며 상속 확률이나 효과는 없다.
- 세대 종결 유형은 5종 enum을 두었지만 자동 흐름에서 실제 발화하는 유형은 `NaturalDeath`, `Extinction` 2종이다.
- 3세대 이상 진행 경로는 구조상 가능하나 검증용 분기로 2세대에서 멸문한다.
- 후계 후보 2명 이상 선택 UI 분기는 코드에 있지만 현재 시드는 후보 1명만 반환한다.
- 사건 문안은 기존 단일 세대 검증 콘텐츠를 재사용한다.
- 가문명, 인물명, 도입/요약/종결 문장은 자리표시자다.
- 완성형 가문 생성기, 저장/불러오기, 혼인/혈통/계보 시뮬레이션, 대량 사건 콘텐츠, 완성형 UI/아트 스타일은 제외 범위 그대로 남아 있다.
