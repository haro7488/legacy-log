# Vertical Slice 1 UI Layout Direction

> 작성일: 2026-05-22
> 상태: 기획 기준

## 목적

이 문서는 Vertical Slice 1 구현 이후 UI 개선 위임을 위해, 화면 구조와 정보 배치 원칙을 제품 기준으로 정리한다.

이번 문서는 구체적인 Godot 노드 구조, 클래스 분리, 데이터 바인딩 방식, 자동 검증 방식은 정하지 않는다. 해당 결정은 Project Codex와 Project Claude의 하위 구현 흐름에 위임한다.

목표는 작은 창과 세로형 화면에서도 플레이어가 현재 가문 상황, 사건 판단, 상태 확인, 연대기 확인, 가문사 누적을 빠르게 오갈 수 있게 하는 것이다.

## 기본 화면 구조

UI는 다음 3단 구조를 기본 방향으로 둔다.

```text
┌──────────────────────────────┐
│ 상단 헤더                     │
│ 가문 · 대표작위 · 세대 · 인물  │
│ 현재 단계 / 화면 목적 / 경고   │
├──────────────────────────────┤
│                              │
│ 중간 게임 화면                │
│ 선택된 탭의 전체 내용          │
│                              │
│ 사건 / 결과 / 세대 시작        │
│ 상태 / 연대기 / 가문사         │
│                              │
├──────────────────────────────┤
│ 하단 탭 메뉴                  │
│ [사건] [상태] [연대기] [가문사] │
└──────────────────────────────┘
```

## 상단 헤더

상단 헤더는 항상 보이는 현재 맥락만 표시한다.

- 현재 가문명
- 대표작위
- 세대
- 현재 인물
- 현재 단계
- 화면 목적
- 작위 위험 경고

작위 위험 경고는 실제 판단이 필요한 단계에서만 강하게 보인다. Stable이나 Watched 같은 낮은 단계의 상세 정보는 상태 화면에서 확인하게 한다.

헤더에는 핵심 상태 4축의 전체 목록, 전체 유산 태그, 세부 수치, 긴 설명을 넣지 않는다.

## 중간 게임 화면

중간 화면은 선택된 하단 탭의 전체 내용을 보여주는 단일 콘텐츠 영역이다.

하단 탭에서 `사건`을 선택하면 다음 화면들이 중간 영역에 표시된다.

- 세대 시작
- 사건
- 결과
- 세대 요약
- 멸문 또는 실행 종료

하단 탭에서 `상태`, `연대기`, `가문사`를 선택하면 해당 정보가 보조 패널이 아니라 중간 화면 전체를 사용해 표시된다.

## 하단 탭 메뉴

하단 탭은 정보 보조 영역이 아니라 주 화면 전환 메뉴다.

기본 탭은 다음 4개로 둔다.

| 탭 | 역할 |
|---|---|
| 사건 | 현재 진행 화면. 세대 시작, 사건, 결과, 세대 요약, 멸문 화면을 포함한다. |
| 상태 | 재산, 명예, 궁정 영향력, 가문 결속, 작위 위험, 현재 인물 특성을 보여준다. |
| 연대기 | 현재 세대의 사건 기록과 최신 기록을 보여준다. |
| 가문사 | 활성 유산 태그, 보유 작위, 지난 세대 이력을 보여준다. |

탭 메뉴는 화면 하단에 고정한다. 작은 창에서도 탭이 본문이나 선택지를 밀어내지 않고, 현재 선택된 화면을 분명히 구분할 수 있어야 한다.

## 사건 화면

사건 화면은 선택 판단에 필요한 정보만 우선한다.

표시 순서는 다음을 권장한다.

1. 사건 진행도
2. 사건 분류
3. 사건 제목
4. 사건 본문
5. 등장 이유 또는 관련 단서
6. 선택지
7. 선택지별 영향 요약

선택지는 별도 하단 ActionBar가 아니라 사건 화면 내부의 주요 행동 영역으로 배치한다. 선택지의 행동명은 짧게 쓰고, 영향 요약은 한 줄 기준으로 제한한다.

## 결과 화면

결과 화면은 선택이 무엇을 남겼는지 확인하는 화면이다.

표시 순서는 다음을 권장한다.

1. 결과 문장
2. 상태 변화
3. 유산 태그 변화
4. 작위 변화 또는 작위 위험 변화
5. 연대기 기록
6. 다음 사건 또는 세대 요약으로 가는 버튼

상태, 유산, 작위, 연대기 중 변화가 없는 항목은 억지로 큰 영역을 차지하지 않는다.

## 상태 화면

상태 화면은 숫자 최적화보다 판단 가능한 말이 먼저 보여야 한다.

기본 표시 항목은 다음이다.

- 재산의 질적 구간
- 명예의 질적 구간
- 궁정 영향력의 질적 구간
- 가문 결속의 질적 구간
- 작위 위험 단계
- 현재 인물의 성향, 강점, 약점

세부 수치나 내부 점수는 접기/펼치기 또는 보조 표시로 둔다. 기본 화면에서는 질적 구간과 변화 방향을 우선한다.

## 연대기 화면

연대기 화면은 현재 세대에서 플레이어가 만든 기록을 확인하는 화면이다.

- 최신 기록은 눈에 띄게 표시한다.
- 큰 사건과 보조 사건을 구분한다.
- 사건 ID나 내부 키는 수동 검증에는 도움이 될 수 있으나, 최종 UI 문체로 확정하지 않는다.

## 가문사 화면

가문사 화면은 여러 세대를 거치며 누적된 결과를 보여준다.

기본 표시 항목은 다음이다.

- 활성 유산 태그
- 유산 태그의 등급 또는 지속 범위
- 유산 태그 한 줄 설명
- 보유 작위
- 지난 세대 이력
- 멸문 또는 쇠락 위험의 누적 이유

유산 태그는 이름만 나열하지 말고, 가능하면 플레이어가 다음 세대 조건과 연결해 읽을 수 있는 설명을 붙인다.

## 수동 검증 질문

UI 개선 후 수동 검증에서는 다음을 확인한다.

- 상단 헤더만 보고 현재 가문, 작위, 세대, 인물, 단계가 이해되는가?
- 하단 탭을 통해 사건, 상태, 연대기, 가문사를 쉽게 오갈 수 있는가?
- 사건 화면에서 선택지와 영향 요약이 화면 안에서 함께 읽히는가?
- 결과 화면에서 상태, 유산, 작위, 연대기 변화가 순서대로 이해되는가?
- 상태 화면에서 숫자를 열지 않아도 현재 가문의 위험과 강점을 판단할 수 있는가?
- 연대기 화면에서 최신 기록과 큰 사건이 구분되는가?
- 가문사 화면에서 이전 세대의 흔적이 현재 세대와 연결되어 보이는가?
- 360x640, 540x720, 1280x720에서 하단 탭과 선택지가 겹치거나 잘리지 않는가?

## Project Codex로 넘길 구현 요구

Project Codex는 이 문서를 바탕으로 UI 개선 구현 계획을 세운다.

계획에는 다음이 포함되어야 한다.

- 기존 UI 구조에서 3단 구조로 전환할 때의 변경 범위
- 사건 탭과 상태/연대기/가문사 탭의 상태 전환 방식
- 선택지와 진행 버튼을 사건 화면 내부로 옮길지 여부와 그 구현 방식
- 작은 창과 세로형 화면의 최소 검증 절차
- 기존 smoke 검증을 유지하면서 UI 수동 검증을 분리하는 보고 방식

Project Codex가 현재 구현 구조와 충돌하는 지점을 발견하면, 임의로 구현하지 말고 Root Codex에 확인 질문으로 되돌린다.
