# Vertical Slice 1 수동 플레이테스트 가이드

> 최종 수정: 2026-05-19

## 목적

이 문서는 Vertical Slice 1 구현 후 사용자가 직접 플레이하며 무엇을 확인하고 어떤 형식으로 피드백을 남길지 정리한다.

Vertical Slice 1의 검증 목표는 기능이 존재하는지 확인하는 데서 끝나지 않는다. 반복 플레이에서 서로 다른 가문사가 만들어지는지, 선택이 의미 있게 느껴지는지, UI/UX가 판단을 돕는지를 함께 검증한다.

## 테스트 전제

- 한 실행은 보통 10~20분 플레이를 목표로 한다.
- 최소 3회 이상 반복 플레이해 서로 다른 양상을 비교한다.
- 작은 창과 세로형 화면에서 최소 1회씩 확인한다.
- 테스트 중에는 기능 오류와 UX 불편을 분리해 기록한다.
- 수치 밸런스는 최종값이 아니므로, 정확한 수치보다 체감과 판단 가능성을 우선 기록한다.

## 피드백 분류 기준

피드백은 다음 다섯 종류로 나누어 기록한다.

| 분류 | 의미 | 예시 |
|---|---|---|
| 기능 오류 | 의도한 기능이 동작하지 않음 | 버튼이 눌리지 않음, 세대가 넘어가지 않음 |
| 정보 부족 | 판단에 필요한 정보가 보이지 않음 | 작위 상실 이유를 알 수 없음 |
| 정보 과다 | 너무 많은 정보가 한꺼번에 나옴 | 태그 목록이 길어 핵심을 못 봄 |
| 재미 부족 | 선택이 의미 없거나 반복이 지루함 | 승격 선택이 항상 정답처럼 보임 |
| 문장/톤 문제 | 문체, 문장 길이, 의미 전달 문제 | 세대 요약이 너무 길거나 감정이 과함 |

## 사용자 피드백 요청 문구

구현 빌드 전달 시 사용자에게 다음 형식으로 피드백을 요청한다.

```text
이번 빌드는 기능 존재 여부만 보지 않고, 반복 플레이 재미와 UI/UX 판단 가능성을 함께 확인하는 빌드입니다.

가능하면 3회 이상 플레이해 주세요.
1회는 일반 창, 1회는 작은 창, 1회는 세로형 화면에 가깝게 확인해 주세요.

각 회차마다 다음을 알려 주세요.
- 몇 세대까지 갔는지
- 어떤 작위로 끝났는지
- 멸문/쇠락/상승이 납득됐는지
- 가장 의미 있었던 선택과 가장 이해하기 어려웠던 선택
- UI에서 자주 본 정보와 찾기 어려웠던 정보
- 다시 플레이하고 싶은지

불편한 점은 기능 오류, 정보 부족, 정보 과다, 재미 부족, 문장/톤 문제 중 어디에 가까운지도 함께 적어 주세요.
```

## Root Codex 후속 판단 기준

수동 검증 후 Root Codex는 다음을 판단한다.

- 콘텐츠 분량을 늘릴지, 사건 조건을 조정할지
- 작위 상승/상실이 재미의 중심으로 작동하는지
- 유산 태그가 다음 세대 체감에 충분한지
- UI 정보 계층화가 작은 창과 세로형 화면에서 버티는지
- 문체 라운드로 넘어갈 준비가 되었는지
- Project Codex에 구현 개선을 다시 위임할지
