# LegacyLog PRD

> 최종 수정: 2026-05-17
> 상태: 초안

## 1. 제품 목표

LegacyLog는 가문 연대기 로그라이크 게임이다. 이 게임은 Godot + C# 기반 원격 개발 워크플로우를 검증하기 위한 실험 매개체이기도 하다.

제품 측면의 목표는 플레이어가 한 인물이 아니라 여러 세대에 걸친 가문 기록을 만들고, 선택과 사건의 누적 결과를 연대기로 확인하는 경험을 제공하는 것이다.

실험 측면의 목표는 루트 PRD와 하위 Godot 구현 프로젝트를 분리해, 텍스트 기반 요구사항 관리와 구현 위임이 실제로 작동하는지 검증하는 것이다.

## 2. 대상 사용자

- 짧은 플레이 세션 안에서 선택의 결과가 누적되는 전략/로그라이크 경험을 원하는 플레이어
- 캐릭터 개인보다 가문, 계보, 연대기, 사건 기록에 관심이 있는 플레이어
- 이 저장소에서는 실제 플레이어 경험뿐 아니라 Codex 원격 개발 워크플로우의 검증 가능성도 주요 대상이다

## 3. MVP 범위

MVP 범위는 아직 확정되지 않았다. 확정 전까지는 다음 항목을 후보로 둔다.

- 새 가문 또는 첫 세대 시작
- 선택 기반 사건 진행
- 선택 결과가 인물 또는 가문 상태에 반영
- 세대 교체 또는 후계 흐름
- 플레이 기록을 연대기 형태로 표시

MVP 확정 전에는 하위 Godot 프로젝트에서 대규모 구현을 시작하지 않는다. 먼저 이 문서에서 MVP 목표와 완료 기준을 좁힌다.

## 4. 핵심 경험 요구사항

- 플레이어는 매 턴 또는 사건마다 의미 있는 선택을 한다.
- 선택은 즉시 결과뿐 아니라 이후 세대 또는 가문 기록에 영향을 준다.
- 게임은 단일 캐릭터의 생존보다 가문의 지속성과 기록을 중심에 둔다.
- 연대기는 단순 로그가 아니라 플레이어가 만든 가문사의 요약으로 읽혀야 한다.

## 5. 기능 요구사항

현재 확정된 구현 요구사항은 없다. 아래는 PRD에서 앞으로 확정해야 할 요구사항 영역이다.

- 시작 흐름: 새 게임, 가문 생성, 초기 인물 설정
- 사건 흐름: 사건 제시, 선택지, 결과 적용
- 상태 모델: 인물 상태, 가문 상태, 세대 정보
- 연대기: 사건 기록, 세대별 요약, 표시 방식
- 실패/종료 조건: 가문 단절, 목표 달성, 기타 종료 조건

## 6. 비범위

초기 PRD 단계에서는 다음을 범위 밖으로 둔다.

- 완성형 UI/아트 스타일 확정
- 밸런스 수치 확정
- 대량 콘텐츠 작성
- 저장/불러오기 세부 구현
- CI, 배포, 플랫폼 패키징

## 7. 구현 연결 방식

루트 폴더는 PRD와 현재 초점을 관리한다. 하위 `legacy-log-project/`는 실제 Godot 구현을 담당한다. 기본 흐름은 `docs/decisions/002-agent-workflow.md`를 따른다.

구현 세션에 작업을 넘길 때는 Root Codex가 `docs/current-focus.md`에 다음을 적는다.

- 참조할 PRD 섹션
- 목적
- 이번 범위
- 제외 범위
- 사전 가정
- 완료 기준
- 보고 형식

이후 하위 흐름은 다음 순서로 진행한다.

1. Project Codex가 작업 지시를 기준으로 아키텍처 및 구현 계획을 작성한다.
2. Project Claude가 구현 계획을 검토하고 개선점을 제안한다.
3. Project Codex가 개선점을 검토한 뒤 최종 작업 지시서를 작성한다.
4. Project Claude가 최종 작업 지시서에 따라 구현한다.

하위 구현 에이전트는 `docs/current-focus.md`와 최종 작업 지시서에서 지정한 범위만 구현 대상으로 삼는다.

## 8. 미결정 사항

- MVP의 정확한 첫 플레이 루프
- 첫 세대 생성 방식
- 사건 데이터의 최소 구조
- 상태 값의 종류와 의미
- 연대기 표시 형식
- 첫 구현 세션의 구체적 목표
