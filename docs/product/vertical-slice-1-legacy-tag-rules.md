## 목적

이 문서는 Vertical Slice 1에서 유산 태그가 얼마나 오래 남고, 상반되는 태그가 어떻게 교체되며, UI와 세대 시작 화면에서 어떤 식으로 설명되어야 하는지 정리한다.

목표는 유산 태그가 단순 장식이 아니라 다음 세대 사건 조건, 세대 시작 상황, 가문사의 정체성을 만드는 기획 매개가 되게 하는 것이다.

## 태그 등급

유산 태그는 지속 기간 기준으로 세 등급을 가진다.

| 등급 | 의미 | 기본 지속 |
|---|---|---|
| 단기 태그 | 현재 세대 또는 다음 사건 몇 개에만 영향을 주는 흔적 | 현재 세대 안에서 해소 |
| 세대 태그 | 다음 세대 시작 상황과 사건 조건에 영향을 주는 유산 | 다음 1세대까지 유지 |
| 가문사 태그 | 여러 세대 동안 가문의 정체성으로 남는 큰 기록 | 2세대 이상 유지 또는 해소 사건 전까지 유지 |

구현 수치나 정확한 턴 수는 이 문서에서 정하지 않는다. 이 문서는 제품 기준만 정한다.

## 태그별 기본 지속 기준

| 태그 | 계열 | 기본 등급 | 해소 또는 교체 기준 |
|---|---|---|---|
| 왕의 총애 | 왕권/궁정 | 세대 태그 | `궁정의 의심`, 큰 궁정 실패, 왕실 요구 거절로 약화 또는 교체 |
| 궁정의 의심 | 왕권/궁정 | 세대 태그 | 공개 충성, 왕실 기여, `왕의 총애` 획득으로 해소 |
| 정적의 표적 | 왕권/궁정 | 가문사 태그 | 정적 방어 성공, 정적 몰락 사건, 작위 상실로 방향 전환 |
| 왕실의 채무 | 왕권/궁정 | 세대 태그 | 왕실 보상 회수, 재산 지불, 후속 청원 성공으로 해소 |
| 전쟁 공훈 | 명예/불명예 | 가문사 태그 | 큰 패배 또는 불명예 사건으로 약화될 수 있으나 쉽게 사라지지 않음 |
| 기사의 명예 | 명예/불명예 | 세대 태그 | 비겁한 선택, 포로 학살, `불명예` 획득으로 교체 |
| 불명예 | 명예/불명예 | 가문사 태그 | 공개 속죄, 큰 전공, 명예 회복 사건으로 약화 |
| 겁쟁이의 소문 | 명예/불명예 | 세대 태그 | 위험 감수 전공, 공개 결투, 영웅적 방어로 해소 |
| 풍요로운 영지 | 영지/재산 | 세대 태그 | 흉작, 반란, 과도한 징발, `피폐한 영지`로 교체 |
| 피폐한 영지 | 영지/재산 | 세대 태그 | 도로 정비, 세금 완화, 풍년/상업 회복으로 해소 |
| 상인 후원 | 영지/재산 | 세대 태그 | 상단 계약 파기, 부채 위기, 궁정 보수파 압력으로 약화 |
| 가문의 부채 | 영지/재산 | 가문사 태그 | 세금 면제, 부채 상환, 영지 부흥으로 약화 |
| 안정된 후계 | 가문 내부/후계 | 세대 태그 | 후계자 사망, 후계권 무효 청원, `불안한 후계`로 교체 |
| 불안한 후계 | 가문 내부/후계 | 세대 태그 | 공개 지명, 왕실 인가, 친족 합의로 해소 |
| 내부 균열 | 가문 내부/후계 | 가문사 태그 | 친족 화해, 방계 포섭, 숙청 성공으로 약화 |
| 충성스러운 친족 | 가문 내부/후계 | 세대 태그 | 권한 남용, 후계 경쟁, 친족 사망으로 소멸 |
| 승격 명분 | 작위/정치적 입지 | 단기 태그 | 승격 청원에 사용되거나 시간이 지나 약화 |
| 불안한 승격 | 작위/정치적 입지 | 세대 태그 | 작위 안정 사건, 궁정 승인, 정적 방어로 해소 |
| 박탈 위기 | 작위/정치적 입지 | 단기 태그 | 방어 성공으로 해소, 실패 시 대표작위 상실 또는 쇠락 |
| 변경의 책무 | 작위/정치적 입지 | 가문사 태그 | 변경 방어 성공으로 공훈화, 실패 시 패배/박탈 위험 |
| 피의 원한 | 비극/몰락 | 가문사 태그 | 화해, 보상, 원한의 결말 사건 전까지 유지 |
| 잊히지 않는 패배 | 비극/몰락 | 가문사 태그 | 큰 승리로 약화될 수 있으나 가문사에는 남음 |
| 가문의 맹세 | 비극/몰락 | 가문사 태그 | 맹세 이행 또는 배신 사건으로 변형 |
| 몰락의 징조 | 비극/몰락 | 세대 태그 | 작위 회복, 후계 안정, 영지 회복으로 해소. 실패하면 멸문 위험 상승 |

## 상반 태그 교체 기준

상반 태그는 동시에 오래 유지하지 않는다. 새 사건이 기존 태그와 명확히 반대되는 결과를 만들면 다음 원칙을 따른다.

1. 새 결과가 더 큰 사건이면 기존 태그를 교체한다.
2. 새 결과가 보조 사건이면 기존 태그를 약화시키고, 즉시 교체하지 않을 수 있다.
3. 두 태그가 모두 가문사적으로 중요하면 하나는 현재 활성 태그, 다른 하나는 과거 기록으로 남긴다.
4. 교체가 발생하면 UI에서 "이전 유산이 약화됨" 또는 "새 유산으로 대체됨"을 설명해야 한다.

대표 상반 관계는 다음과 같다.

| 태그 A | 태그 B | 처리 기준 |
|---|---|---|
| 왕의 총애 | 궁정의 의심 | 최근 큰 궁정 사건 결과를 우선한다. |
| 기사의 명예 | 불명예 | 큰 명예/불명예 사건이 서로를 교체하거나 약화한다. |
| 풍요로운 영지 | 피폐한 영지 | 영지 상태를 대표하는 현재 태그는 하나만 활성화한다. |
| 안정된 후계 | 불안한 후계 | 현재 후계 구도의 상태를 하나만 활성화한다. |
| 승격 명분 | 박탈 위기 | 승격 준비와 박탈 위기는 동시에 오래 유지하지 않는다. 최근 작위 사건을 우선한다. |
| 충성스러운 친족 | 내부 균열 | 친족 관계가 변하면 하나는 활성, 하나는 과거 기록으로 밀린다. |

## 태그 노출 기준

UI에서 태그는 세 곳에 노출된다.

### 사건 화면

사건이 특정 태그 때문에 등장했거나 강화되었다면, 사건 화면에서 관련 태그를 짧게 보여준다.

예:

- 관련 유산: `정적의 표적`
- 이 사건은 이전 세대의 궁정 갈등 때문에 발생했다.

### 결과 화면

선택 결과로 태그가 추가, 약화, 교체, 해소되면 결과 화면에서 바로 보여준다.

예:

- 새 유산: `왕의 총애`
- 약화된 유산: `궁정의 의심`
- 교체: `피폐한 영지` -> `풍요로운 영지`

### 세대 시작 화면

세대 시작 화면에는 현재 세대에 가장 큰 영향을 줄 태그 2~4개를 우선 노출한다.

우선순위는 다음과 같다.

1. 작위 상승/상실과 연결된 태그
2. 후계 안정과 연결된 태그
3. 현재 사건 풀을 바꾸는 태그
4. 가문사 정체성을 보여주는 장기 태그

## 태그 적용 원칙

- 모든 사건이 태그를 남길 필요는 없다.
- 큰 사건은 0~2개 태그를 남길 수 있다.
- 보조 사건은 보통 0~1개 태그를 남긴다.
- 한 세대에 활성 태그가 너무 많아지면, UI는 핵심 태그만 보여주고 나머지는 가문사 상세로 보낸다.
- 태그는 선택의 결과를 설명해야 하며, 플레이어가 이해하지 못하는 숨은 조건으로만 쓰면 안 된다.
- 태그는 다음 세대 사건 조건에 쓰일 때 반드시 세대 시작 또는 사건 화면에서 이유가 드러나야 한다.

## 사건 연결 예시

| 사건 | 태그 변화 예시 | 다음 영향 |
|---|---|---|
| EST-01 흉작의 해 | `피폐한 영지` 추가 또는 해소 | 영지 반란, 탄원, 재산 사건 우선도 변화 |
| CRT-05 밀서의 발견 | `왕의 총애` 또는 `불명예` | 승격/궁정 사건 또는 정적 사건 조건 |
| FAM-08 후계자 간 경쟁 | `안정된 후계` 또는 `불안한 후계` | 세대 시작 안정도, 후계권 무효 청원 조건 |
| PRO-03 변경 방위 공적 | `변경의 책무` 추가 | 후작 승격, 전쟁/변경 사건 우선도 증가 |
| CRI-04 영지 반란 | `피의 원한`, `불명예`, `피폐한 영지` | 쇠락/내부 균열/영지 사건 강화 |
| END-04 강제 폐위의 밤 | `몰락의 징조`, `피의 원한` | 다음 세대 시작 압력, 멸문 위험 증가 |

## 남은 Root 기획 작업

- 사건별 구체 태그 추가/해소 표 작성
- 세대별 활성 태그 최대 노출 개수 확정
- 태그별 세대 시작 문장 템플릿 작성
- 작위 상승/상실 조건과 태그 조건 연결
- 세대별 사건 풀 선택 기준 작성
