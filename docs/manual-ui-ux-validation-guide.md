# 다세대 MVP UI/UX 수동 검증 가이드

> 작성일: 2026-05-19
> 대상: 사용자 수동 플레이 검증

## 목적

이 문서는 사용자가 다세대 MVP를 직접 플레이하면서 기능 동작뿐 아니라 UI/UX 편의성을 함께 확인하기 위한 안내서다.

검증자는 "기능이 된다/안 된다"만 보지 않고, 화면을 읽고 조작하는 과정이 이해 가능한지, 불편한 지점이 어디인지, 다음 구현 작업으로 되돌릴 피드백이 무엇인지 함께 기록한다.

## 현재 구현된 것

하위 handoff 보고 기준으로 현재 구현된 기능은 다음과 같다.

- 새 가문 실행 시작
- 1세대 인물로 사건 3개 진행
- 사건별 선택지 선택
- 선택 결과 문구 표시
- 상태 변화 표시
- 진행 중 연대기 항목 누적
- 1세대 세대 요약 표시
- 후계 후보 확인
- 후계 선택 또는 다음 세대 진입
- 2세대 사건 3개 진행
- 2세대 종료 후 후계 부재와 사망/퇴장 조건에 따른 멸문
- 가문 전체 이력 표시
- `가문력 N년` 형식의 시간 표시
- 상태의 4단계 질적 라벨 표시
- 상세 상태 토글에서 숫자와 변화 방향 표시
- 표시 전용 특성 스텁
- 가문명, 인물명, 도입/요약/종결 문장의 자리표시자화

## 실행 전 전제

- 이번 검증의 목표는 완성형 UI/아트 평가가 아니다.
- 자리표시자 문구와 임시 라벨은 최종 문안이 아니다.
- 핵심은 플레이 흐름, 정보 이해 가능성, 조작 편의성, 화면별 다음 행동의 명확성이다.

## 실행 방법

하위 Godot 프로젝트에서 실행한다.

```powershell
cd legacy-log-project
.\scripts\godot.cmd
```

실행이 되지 않으면 실행 오류 메시지와 실행한 명령을 그대로 기록한다.

## 테스트할 사용자 흐름

아래 순서대로 직접 플레이한다.

1. 새 실행을 시작한다.
2. 시작 화면에서 가문, 인물, 현재 목표를 이해할 수 있는지 확인한다.
3. 1세대 사건 1을 읽고 선택지를 고른다.
4. 선택 결과 화면에서 결과 문구, 상태 변화, 연대기 기록을 확인한다.
5. 상세 상태 토글을 켜고 끄며 숫자와 변화 방향이 이해되는지 확인한다.
6. 1세대 사건 2와 사건 3을 진행한다.
7. 1세대 세대 요약 화면을 확인한다.
8. 후계 화면에서 다음 세대로 넘어가는 이유와 버튼 흐름을 확인한다.
9. 2세대 진입 화면에서 이전 세대의 흔적이 보이는지 확인한다.
10. 2세대 사건 1-3을 진행한다.
11. 2세대 세대 요약과 멸문 화면을 확인한다.
12. 멸문 화면에서 가문 전체 이력과 게임 종료 상태가 명확한지 확인한다.

## 화면별 확인 항목

### 시작 화면

- 이 게임이 가문 단위 진행이라는 점이 보이는가?
- 현재 인물이 누구인지 알 수 있는가?
- 첫 행동 버튼이 명확한가?
- 자리표시자 문구가 최종 설정처럼 오해되지 않는가?

### 사건 화면

- 현재 몇 세대인지 알 수 있는가?
- 현재 인물과 가문 상태를 헷갈리지 않는가?
- 사건 본문과 선택지가 읽기 편한가?
- 선택지 버튼 위치와 문구가 자연스러운가?

### 결과 화면

- 내가 고른 선택의 결과가 바로 보이는가?
- 상태 변화가 어떤 의미인지 최소한 이해되는가?
- 연대기 기록과 결과 문구의 차이가 보이는가?
- 다음 사건으로 넘어가는 행동이 명확한가?

### 상세 상태 토글

- 상세 토글을 찾을 수 있는가?
- 기본 상태 라벨과 상세 숫자의 관계가 이해되는가?
- 상세 정보가 너무 많거나 기본 흐름을 방해하지 않는가?

### 세대 요약 화면

- 한 세대가 끝났다는 감각이 드는가?
- 이번 세대가 남긴 유산이 보이는가?
- 다음 세대로 이어지는 상황 암시가 보이는가?
- 누적 연대기와 최종 상태 요약이 구분되는가?

### 후계 화면

- 왜 후계로 넘어가는지 이해되는가?
- 후계 후보 정보가 충분한가?
- 다음 세대로 가는 버튼이 명확한가?
- 후계 선택이 자동인지 선택인지 헷갈리지 않는가?

### 2세대 진입 화면

- 이전 세대의 상태, 사건 결과, 기록이 이어졌다는 점이 보이는가?
- 1세대와 2세대의 구분이 명확한가?
- 같은 사건이 반복되어도 검증용 반복이라는 점이 크게 방해되지 않는가?

### 멸문 화면

- 가문이 끝났다는 사실이 분명한가?
- 왜 멸문했는지 이해되는가?
- 전체 세대 이력을 되돌아볼 수 있는가?
- 다시 시작하거나 종료해야 하는 다음 행동이 명확한가?

## 피드백 기록 형식

검증 후 아래 형식으로 피드백을 남긴다.

```markdown
## 수동 UI/UX 검증 피드백

### 실행 결과
- 실행 가능 여부:
- 끝까지 플레이 가능 여부:
- 멈춘 화면 또는 오류:

### 이해하기 어려웠던 화면
- 화면:
- 무엇이 헷갈렸는가:
- 기대한 정보 또는 행동:

### 조작이 불편했던 지점
- 화면:
- 불편했던 버튼/토글/흐름:
- 개선되면 좋을 방향:

### 정보가 부족하거나 과했던 지점
- 부족했던 정보:
- 너무 많거나 읽기 어려웠던 정보:
- 우선순위를 바꿔야 할 정보:

### 세대 흐름 체감
- 1세대에서 2세대로 이어진다는 느낌이 있었는가:
- 이전 세대의 유산이 보였는가:
- 멸문이 납득 가능했는가:

### 상태 표시 피드백
- 질적 라벨이 이해됐는가:
- 상세 숫자/변화 방향이 필요할 때 찾을 수 있었는가:
- 상태 표시가 선택 판단에 도움이 됐는가:

### 종합 판단
- 기능은 정상적으로 보였는가:
- UI/UX는 다음 구현으로 넘겨도 될 수준인가:
- 반드시 고쳐야 할 문제:
- 있으면 좋은 개선:
```

## Project Codex로 되돌릴 기준

다음 중 하나라도 있으면 개선 작업으로 되돌린다.

- 다음에 눌러야 할 버튼을 찾기 어렵다.
- 현재 세대와 이전 세대가 헷갈린다.
- 선택 결과와 연대기 기록의 차이를 이해하기 어렵다.
- 세대 요약에서 유산과 다음 세대 상황이 보이지 않는다.
- 후계 또는 멸문 이유를 이해하기 어렵다.
- 상태 라벨과 상세 숫자의 관계가 이해되지 않는다.
- 텍스트가 너무 많거나 잘려서 읽기 어렵다.
- 멸문 화면에서 게임 종료와 전체 이력이 충분히 전달되지 않는다.

## 검증 완료 판단

수동 검증 완료는 다음을 모두 만족할 때로 본다.

- 사용자가 시작부터 멸문까지 직접 플레이했다.
- 각 주요 화면의 이해 가능성과 조작 흐름을 확인했다.
- 불편한 지점이 없거나, 발견된 불편을 피드백 형식으로 기록했다.
- 기능 완료와 UI/UX 완료 여부를 분리해서 판단했다.
