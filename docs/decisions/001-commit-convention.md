# 001. 커밋 메시지 관례

## 상태
확정 (2026-04-11)

## 맥락

프로젝트 초반 커밋 3개를 만든 뒤, 메시지 스타일이 일관되지 않음을 발견했다:

- `2ed2bee` — `Initial commit: Godot C# project + docs structure` (자유 형식)
- `6d52a1b` — `Add Obsidian vault config and initial setup log` (자유 형식)
- `3026092` — `docs: add root CLAUDE.md defining meta agent scope` (Conventional Commits)

세 번째 커밋에서 무의식적으로 `docs:` 접두사를 쓰기 시작했고, 이 시점에 "앞으로의 관례를 명시할지" 결정이 필요해졌다.

이 프로젝트는 제품 문서, 결정 문서, 구현 작업이 함께 누적된다. 커밋 메시지가 분류 가능한 형식을 따르면, 나중에 `git log --grep` 같은 방식으로 "문서 변경만", "설정 변경만", "게임 기능 변경만" 따로 뽑아볼 수 있다.

## 고려한 대안

- **(가) Conventional Commits 통일** — 모든 커밋 메시지에 정해진 접두사(`feat:`, `fix:`, `docs:`, `chore:` 등)를 붙인다. 장점: 자동 분류·필터링 가능, 업계 표준, 이미 3026092에서 시작됨. 단점: 접두사 목록을 기억해야 하는 작은 부담.

- **(나) 자유 형식 유지** — 개인 프로젝트이므로 관례에 묶이지 않고 "뭘 했는지만 이해되면 충분"하다는 입장. 장점: 쓰기 편함, 과거 두 커밋과 일관됨. 단점: 로그 분석·필터링이 전수 검색에 의존, 스타일이 점점 표류할 가능성.

## 결정

**Conventional Commits 관례를 채택한다.** 앞으로의 모든 커밋 메시지는 다음 형식을 따른다:

```
<type>: <짧은 설명>
```

사용할 `type` 접두사 목록:

| 접두사 | 용도 |
|---|---|
| `feat:` | 게임 기능·코드 추가 |
| `fix:` | 버그 수정 |
| `docs:` | 문서 추가·수정 (CLAUDE.md, decisions, README 등) |
| `chore:` | 설정·인프라 (Git, Obsidian, CI 등) |
| `refactor:` | 코드·문서 구조 개선 |

접두사(`type`)는 영어 소문자로 고정한다. 설명은 한국어 또는 영어 중 편한 쪽을 사용하되, **마침표 없음**과 **간결함**을 지킨다. 예: `docs: 커밋 관례 결정 문서 추가` 또는 `docs: add commit convention decision`.

본문이나 푸터는 선택적이다. 필요할 때만 추가한다.

## 근거

**(가)를 선택한 두 가지 주된 이유:**

1. **변경 이력 분석의 가치.** 이 저장소는 PRD, 결정 문서, Godot 구현이 함께 바뀐다. 커밋 로그가 분류 가능하면 나중에 "문서 변경", "설정 변경", "게임 기능 변경"을 쉽게 구분할 수 있다. 자유 형식으로는 이게 어려워진다.

2. **관례 정착의 타이밍.** `3026092`에서 `docs:` 접두사를 이미 썼기 때문에, 이 결정을 내리지 않으면 "한 번 쓰고 흐지부지된 관례"로 남는다. 지금 명시적으로 확정하거나, 아니면 의식적으로 자유 형식으로 돌아가거나 둘 중 하나여야 한다. 정착 쪽이 장기 관리에 더 유리하다.

**(나)를 선택하지 않은 이유:**

(나)의 장점인 "쓰기 편함"은 실제로 크지 않다. 접두사 6개를 기억하는 것은 인지 부담이 거의 없다. 반면 (나)의 단점인 "스타일 표류"는 시간이 갈수록 누적된다.

**혼용(접두사 영어 + 설명 자유) 선택 이유:**

접두사는 `git log --grep="^docs:"` 같은 시스템 필터링에 쓰이므로 영어 고정이 자연스럽다. 반면 설명은 사람이 읽는 영역이며, 6개월 후 커밋 히스토리를 훑을 때 모국어가 스캔 속도에서 유리하다. 두 영역의 목적이 다르므로 언어를 분리하는 것이 편의와 기능을 모두 살린다.

**과거 커밋 재작성하지 않는 이유:**

기존 `2ed2bee`와 `6d52a1b`는 그대로 둔다. 히스토리 재작성(`git rebase -i`, 강제 푸시)은 되돌리기 비용이 매우 높은 작업이고, 이 결정의 이득이 그 비용을 정당화하지 못한다. 이 관례는 **001번 결정 문서 이후의 커밋에만** 적용한다.

## 결과 / 적용 범위

- **적용 대상:** 이 저장소의 앞으로 모든 커밋 (로컬·원격 양쪽).
- **영향 받는 파일:** 없음 (관례이지 코드가 아님).
- **예외:** 기존 커밋 `2ed2bee`, `6d52a1b`는 자유 형식 그대로 유지. 재작성하지 않음.
- **검증 방법:** 현재는 수동(사람이 커밋 시 의식적으로). 나중에 필요해지면 commit-msg hook으로 자동 검증 가능하지만, 지금은 도입하지 않는다 (과잉).

## 관련

- 루트 `CLAUDE.md` — 이 관례에 대한 참조 링크가 추가될 예정 (다음 커밋에서).
- 되돌리기 비용이 큰 작업을 피한다는 운영 원칙 — 과거 커밋을 재작성하지 않는 판단의 근거.
