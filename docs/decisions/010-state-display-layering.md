# 010. 상태 표시의 계층화 원칙

## 상태
확정 (2026-05-18)

## 맥락

다세대 MVP에서는 상태가 세대 간 이어지고, 일부는 누적되며, 일부는 감쇠하거나 재구성된다. 플레이어는 선택의 결과를 이해해야 하지만, 모든 숫자와 계산을 항상 노출하면 연대기 경험이 약해질 수 있다.

이번 결정은 상태가 플레이어에게 어떻게 보이는지에 대한 제품 원칙을 정한다. 구체 UI 레이아웃, 상태 이름, 라벨 문구, 수치 임계값, 데이터 구조는 이 문서에서 확정하지 않는다.

## 결정

### 기본 표시

기본 상태 표시는 **질적 구간 라벨 + 세대 끝 서술 요약**으로 한다.

플레이어가 기본 화면에서 먼저 보는 것은 정밀 숫자가 아니라, 현재 가문 상태를 이해할 수 있는 추상화된 상태 표현이다.

### 구간 라벨 단계 수

질적 구간 라벨은 **4-5단계**를 권고한다.

정확한 라벨 문구는 시대/세계관 톤 결정 후 별도 라운드에서 정한다.

### 추가 정보창

추가 정보창에서는 숫자와 변화 방향을 노출한다.

기본 화면에서 추상화된 상태를 본 플레이어가 더 깊이 보고 싶을 때, 현재 값과 변화 방향을 확인할 수 있어야 한다.

### 표시 원칙

상태 표시 원칙은 **정보 계층화**다.

캐주얼한 시선에는 추상화된 상태를 제공하고, 깊이 보고 싶은 사용자에게는 디테일을 제공한다.

## 근거

질적 구간 라벨은 플레이어가 숫자를 해석하지 않아도 가문 상태를 이해하게 한다. 세대 끝 서술 요약은 상태 변화를 연대기와 연결해, 선택의 결과가 단순 수치 변화가 아니라 가문사의 일부처럼 보이게 한다.

4-5단계는 너무 거칠지 않으면서도 플레이어가 빠르게 이해할 수 있는 범위다. 2-3단계는 변화 감각이 약하고, 과도하게 많은 단계는 사실상 숫자 표시와 비슷한 부담을 만든다.

추가 정보창의 숫자와 변화 방향은 전략적 판단을 원하는 사용자를 위한 것이다. 숫자를 완전히 숨기면 선택 결과의 이해와 검증이 약해질 수 있다.

정보 계층화는 연대기 감성과 게임적 판단을 함께 유지하기 위한 방식이다.

## 영향

`docs/decisions/008-multigen-mvp-scope.md`의 재산형, 평판형, 관계형 상태는 기본 화면에서 질적 라벨로 표현될 수 있어야 한다.

상태 수치, 감쇠율, 구간 임계값은 필요하더라도 기본 노출 정보가 아니다. 기본 화면은 추상화된 상태와 세대 끝 요약을 우선한다.

하위 구현 흐름은 추가 정보창을 고려할 수 있지만, 구체 UI 레이아웃과 디자인은 이 결정에서 확정하지 않는다.

## 결정 보류

결정 보류 — 라벨 문구 — 시대/세계관 톤 결정 라운드 위임: 질적 구간 라벨의 실제 명칭은 이 문서에서 정하지 않는다.

결정 보류 — 구체 수치 — 하위 구현 흐름 위임: 상태 값 범위, 감쇠율, 구간 임계값, 변화량 표시 방식은 이 문서에서 정하지 않는다.

결정 보류 — UI 상세 — 하위 구현 흐름 위임: 추가 정보창의 위치, 레이아웃, 조작 방식은 이 문서에서 정하지 않는다.
