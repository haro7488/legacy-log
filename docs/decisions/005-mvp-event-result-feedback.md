# 005. MVP 사건 결과 피드백

## 상태
확정 (2026-05-18)

## 맥락

MVP 첫 플레이 루프는 사건 선택 중심으로 확정되었다. 시작 전제도 고정된 첫 가문과 초대 인물로 정했다. 다음으로 사건 선택의 결과가 플레이어에게 어떤 방식으로 전달되어야 하는지 정해야 한다.

선택 결과가 서사 문장만 제공되면 연대기 감성은 강하지만 게임적 피드백이 약해질 수 있다. 반대로 상태 변화 중심으로 가면 첫 MVP에서 상태 모델 설계 부담이 커진다.

## 결정

MVP의 사건 선택 결과는 **서사 결과 + 연대기 항목 + 최소 상태 변화**를 함께 제공한다.

플레이어는 선택 후 다음을 확인할 수 있어야 한다.

- 선택 직후의 결과 문구
- 연대기에 누적되는 기록 문장
- 선택의 영향을 이해할 수 있는 최소한의 상태 변화

상태 변화는 MVP 루프 검증에 필요한 만큼만 보여준다. 구체적인 상태 값 이름, 개수, 데이터 구조, 구현 방식은 Root Codex가 확정하지 않고 Project Codex와 Project Claude의 하위 흐름에 위임한다.

## 근거

이 방식은 LegacyLog의 핵심 감성인 "선택이 기록으로 남는다"를 유지하면서도, 플레이어가 자신의 선택이 게임 상태에 영향을 줬다는 피드백을 받을 수 있게 한다.

서사 결과만 제공하면 게임 루프 검증이 약하고, 상태 변화 중심으로 가면 첫 MVP 범위가 커진다. 따라서 첫 MVP에서는 둘 사이의 최소 결합을 검증한다.
