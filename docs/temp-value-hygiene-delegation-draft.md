# 임시값 위생 작업 위임 지시서 초안

## 목적

다세대 게임 정체와 시대/세계관 톤이 아직 확정되지 않은 상태에서, 현재 검증용 임시값이 제품 설정처럼 굳어지는 것을 방지한다.

## 수정 범위

- `legacy-log-project/scripts/MvpLoopContent.cs`

대상은 다음에 한정한다.

- 가문명
- 인물명
- 도입 문장
- 세대 마무리 문장

## 변경 방향

현재 임시값인 `하서가`, `하서 단` 및 특정 세계관을 암시할 수 있는 도입/마무리 문장을 명확한 자리표시자형 표현으로 교체한다.

예시는 다음과 같다.

- `가문 A`
- `인물 1`
- `[자리표시자: 도입 문장]`
- `[자리표시자: 세대 마무리 문장]`

정확한 문구는 Project Codex가 기존 UI 길이와 표시 맥락을 확인한 뒤, 자리표시자임이 분명한 형태로 정한다.

## 제외 범위

- 다세대 MVP 메커니즘 구현
- 시대/세계관 톤 확정
- 가문/인물 이름 후보 확정
- 사건 구조 변경
- 상태 값 이름, 개수, 수치, 밸런스 변경
- Godot 씬, 리소스, C# 클래스 구조 변경

## 사전 가정

- `legacy-log-project/scripts/MvpLoopContent.cs`의 해당 값들은 검증용 임시 콘텐츠다.
- 현재 작업은 제품 문안 작성이 아니라 임시값이 확정 설정처럼 보이지 않게 하는 위생 작업이다.
- 실제 코드 변경은 Project Codex의 계획과 Project Claude의 구현 흐름을 따른다.

## 제품 완료 기준

- 화면에 표시되는 가문명과 인물명이 특정 세계관 이름처럼 보이지 않는다.
- 도입 문장과 세대 마무리 문장이 제품 문안 후보가 아니라 자리표시자임을 명확히 드러낸다.
- 단일 세대 루프의 동작 의미는 유지된다.
- 다세대 메커니즘, 세계관 톤, 이름 후보는 새로 확정하지 않는다.

## 보고 형식

- 수정 대상 값 목록
- 자리표시자 문구 제안
- Project Claude에 넘길 최종 작업 지시서
- 범위 밖으로 남긴 항목
