# Current Focus

> 최종 수정: 2026-05-18

## 현재 초점

MVP 첫 플레이 루프를 하위 구현 흐름으로 넘길 수 있도록 제품 요구, 범위, 제외 범위, 사전 가정, 완료 기준을 정리한다.

현재 제품 방향은 **고정된 첫 가문과 초대 인물로 시작하는 3개 사건 선택 루프 + 연대기 출력 + 세대 요약**이다. 플레이어는 사건을 읽고 선택지를 고르며, 선택 결과는 결과 문구, 최소 상태 변화, 연대기 항목으로 피드백된다.

## 목적

첫 MVP 사건 선택 루프를 플레이어가 처음부터 끝까지 진행할 수 있게 만든다.

이 작업의 목적은 완성형 게임 시스템을 만드는 것이 아니라, "선택이 결과와 상태 변화로 피드백되고, 그 결과가 가문 연대기로 누적된다"는 LegacyLog의 핵심 경험을 검증하는 것이다.

## 사용자 경험 범위

- 플레이어는 새 실행을 시작한다.
- 플레이어는 고정된 첫 가문과 초대 인물의 시작 정보를 본다.
- 플레이어는 3개의 사건을 순서대로 진행한다.
- 각 사건은 제목 또는 짧은 본문과 2개 이상의 선택지를 제공한다.
- 플레이어는 선택 후 결과 문구를 확인한다.
- 플레이어는 선택 후 최소 상태 변화 표시를 확인한다.
- 플레이어는 선택 결과가 연대기 항목으로 누적되는 것을 확인한다.
- 3개의 사건이 끝나면 세대 요약을 확인한다.
- 세대 요약은 누적 연대기, 최종 상태 요약, 짧은 마무리 문장을 포함한다.

## 제외 범위

- 완성형 가문 생성기
- 첫 가문과 초대 인물 이름의 최종 확정
- 첫 MVP 사건들의 최종 문안 확정
- 복잡한 후계, 혼인, 혈통 시스템
- 랜덤 사건 선택
- 저장/불러오기
- 밸런스 수치 확정
- 완성형 UI/아트 스타일 확정
- CI, 배포, 플랫폼 패키징

## 사전 가정

- Root Codex는 제품 요구 수준만 정리한다.
- 구체적인 아키텍처, 데이터 저장 형식, 코드 구조, Godot 씬/리소스 구성은 Project Codex와 Project Claude의 하위 흐름에서 결정한다.
- 상태 값 이름, 개수, 수치 범위는 하위 구현 흐름에서 정한다.
- 첫 콘텐츠의 최종 문안은 아직 확정하지 않는다. 구현 검증에 필요한 고정 문안이 필요하면 최종 콘텐츠가 아닌 MVP 검증용 문안으로 취급한다.
- 위 가정 중 하나라도 어긋나면 추측으로 진행하지 않고 사용자 확인을 받는다.

## 제품 완료 기준

- 새 실행을 시작할 수 있다.
- 고정된 첫 가문과 초대 인물의 시작 정보를 볼 수 있다.
- 3개의 사건을 순서대로 진행할 수 있다.
- 각 사건에서 2개 이상의 선택지를 고를 수 있다.
- 선택 후 결과 문구와 최소 상태 변화를 볼 수 있다.
- 선택 결과가 연대기 항목으로 누적된다.
- 3개의 사건 후 세대 요약에서 누적 연대기, 최종 상태 요약, 짧은 마무리 문장을 볼 수 있다.
- Project Claude가 지정된 확인 명령을 실행하고 결과를 보고할 수 있다.

## 보고 형식

Project Codex는 다음 형식으로 결과를 돌려준다.

- 구현 계획 요약
- 수정 또는 생성이 필요한 영역
- 하위 구현에서 정해야 할 상태 모델과 데이터 구조 제안
- 사전 가정 위반 여부
- Project Claude에게 넘길 최종 작업 지시서 초안

Project Claude는 최종 구현 후 다음 형식으로 결과를 돌려준다.

- 변경 파일
- 구현된 사용자 흐름
- 실행한 확인 명령과 결과
- 남은 제약 또는 미구현 범위
