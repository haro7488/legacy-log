# Current Focus

> 최종 수정: 2026-05-18

## 현재 초점

MVP 첫 플레이 루프는 기초 동작 확인까지 완료됐다. 자동 검증과 수동 GUI 확인 기준으로 새 실행, 3개 사건 진행, 선택 결과, 최소 상태 변화, 연대기 누적, 세대 요약은 작동한다.

다만 현재 구현은 **가장 기초적인 MVP 루프가 동작하는 상태**일 뿐이다. UX와 게임 경험 측면에서는 아직 개선이 필요하며, "게임으로서 제대로 기능한다"고 판단하기에는 이르다. 다음 Root Codex 초점은 구현 확장이 아니라 MVP 첫 루프의 기획적 보완 방향을 정리하는 것이다.

## 현재 상태 판단

- 기초 MVP 루프 구현은 완료로 보고할 수 있다.
- Project Claude 검증 명령은 성공으로 보고됐다.
- 사용자 수동 GUI 확인도 완료로 보고할 수 있다.
- 그러나 UX, 선택의 재미, 게임적 목표감, 반복 플레이 동기, 정보 표시 방식은 아직 제품 수준 판단이 필요하다.
- 현재 콘텐츠와 상태 이름, 상태 수치, 가문/인물 이름, 사건 문안은 MVP 검증용 임시 값으로 본다.

## 다음 논의 목적

MVP 첫 루프를 "작동하는 프로토타입"에서 "게임으로 판단 가능한 첫 경험"으로 끌어올리기 위해 필요한 기획 기준을 정한다.

Root Codex는 다음 논의에서 제품 요구 수준만 정리한다. 구체적인 아키텍처, 데이터 구조, Godot 씬 구성, C# 구현 방식은 하위 구현 흐름에 위임한다.

## 논의해야 할 기획 질문

- 플레이어가 첫 루프에서 무엇을 목표로 선택해야 하는가?
- 선택지가 단순 문장 선택을 넘어 의미 있게 느껴지려면 어떤 차이를 보여줘야 하는가?
- 상태 항목은 플레이어에게 어떤 판단 기준을 제공해야 하는가?
- 연대기는 어떤 문체와 정보량이어야 "가문사"처럼 읽히는가?
- 세대 요약은 단순 종료 화면이 아니라 어떤 감정을 남겨야 하는가?
- 첫 가문과 초대 인물은 임시 이름을 유지할지, 제품 문안 후보 정리 단계로 넘길지 결정해야 하는가?
- 다음 작업을 UX 개선, 콘텐츠 문안 정리, 상태 모델 제품 기준 정리 중 어디에 둘 것인가?

## 제외 범위

- 구현 파일 수정
- 하위 프로젝트 내부 문서 수정
- PRD에 없는 게임 규칙 확정
- 상태 값 이름, 개수, 수치, 밸런스 확정
- 사건 데이터 저장 형식 확정
- Godot 씬, 리소스, C# 클래스 구조 결정
- 완성형 UI/아트 스타일 확정
- 저장/불러오기, 랜덤 사건, 후계/혼인/혈통 시스템 구현 지시

## Root Codex 완료 기준

- MVP 첫 루프의 현재 상태를 "기초 동작 완료, 게임 경험은 추가 기획 필요"로 정리한다.
- UX와 게임성 측면에서 부족한 항목을 제품 요구 수준으로 분류한다.
- 다음 하위 구현 흐름으로 넘기기 전에 결정해야 할 기획 질문을 좁힌다.
- 필요한 경우 PRD 또는 결정 문서에 반영할 후보를 사용자와 합의한다.

## 보고 형식

Root Codex는 다음 형식으로 결과를 돌려준다.

- 현재 상태 요약
- UX/게임 경험상 부족한 점
- 추가 기획이 필요한 질문
- PRD 또는 결정 문서 갱신 필요 여부
- 다음 작업 초점 후보
